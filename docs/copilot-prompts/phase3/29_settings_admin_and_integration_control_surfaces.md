# Prompt 29 — Settings, Admin, and Integration Control Surfaces

Finish the admin surfaces so they become trustworthy operating controls instead of mostly informational pages.

## Goals

- make settings and admin pages more useful
- clearly expose current integration and configuration reality
- prepare the app for future operational configuration flows

## Tasks

1. Review the current roles and integrations settings pages.
2. Improve these surfaces to show:
   - role and permission clarity
   - integration status and maturity
   - environment or runtime notes where helpful
   - current limitations and next actions for admin users
3. Make these pages feel like real operational control surfaces, not placeholders.
4. Add or update:
   - `/docs/architecture/admin_surface_model.md`
   - `/docs/architecture/integration_control_status.md`

## Rules

- do not invent fake integrations or fake write capabilities
- keep the pages honest and useful
- preserve the premium shell language

## Deliverable

- stronger settings/admin surfaces
- clearer integration control presentation
- docs for admin and integration status
