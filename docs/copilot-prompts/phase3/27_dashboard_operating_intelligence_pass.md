# Prompt 27 — Dashboard Operating Intelligence Pass

Make the dashboards feel like a true operating intelligence layer instead of a collection of summary cards.

## Goals

- improve role-specific usefulness
- better reflect workstation, approvals, events, and entity runtime state
- prepare dashboards to become real command surfaces

## Tasks

1. Review dashboard behavior for owner, gm, sales, and service contexts.
2. Improve metric derivation and card grouping so dashboards surface:
   - work requiring action now
   - approvals needing resolution
   - aging or risk signals
   - activity and throughput signals
3. Improve role-specific relevance without making dashboards noisy.
4. Add or update:
   - `/docs/ux/dashboard_operating_intelligence.md`

## Rules

- do not build a giant BI platform in this pass
- keep dashboards premium, readable, and operationally sharp
- favor decisions and urgency over decorative metrics

## Deliverable

- stronger operating dashboards
- better role-specific relevance
- docs for dashboard intelligence direction
