# Prompt 25 — Record Linkage and Cross-Surface Continuity

Strengthen how records connect across pages, workstation cards, approvals, events, and dashboards.

## Goals

- make the product feel like one operating system instead of separate screens
- improve linked-record visibility and navigation
- preserve canonical object continuity across surfaces

## Tasks

1. Review record pages, workstation cards, dashboards, approval surfaces, and command palette results.
2. Improve linked-object continuity for at least:
   - lead ↔ household
   - lead ↔ deal
   - deal ↔ inventory
   - approval ↔ linked entity
   - workstation card ↔ linked object
   - event rows ↔ linked object
3. Add better linked-record badges, actions, and navigation where appropriate.
4. Add or update:
   - `/docs/architecture/cross_surface_continuity.md`
   - `/docs/ux/linked_record_behavior.md`

## Rules

- do not invent duplicate entity references
- do not make cross-links feel noisy or cluttered
- prioritize operational clarity

## Deliverable

- stronger cross-surface continuity
- better linked-record behavior
- docs for record linkage model
