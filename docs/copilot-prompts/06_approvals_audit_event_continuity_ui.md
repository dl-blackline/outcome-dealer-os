Finish the approval, audit, and event continuity layer in Outcome Dealer OS surfaces.

## Goals

- make approvals, audit, and event stream visible and coherent in the UI
- make the app reflect its north-star architecture
- ensure trust-sensitive flows are visibly governed

1. Improve Approval Queue, Audit Explorer, and Event Explorer so they feel real and useful.
2. Add reusable UI components for:
   - approval status
   - audit row rendering
   - event row rendering
   - requires-review flags
3. Surface approval and audit summaries on relevant record pages and workstation cards where possible.
4. Add docs:
   - `/docs/architecture/ui_governance_surfaces.md`

## Rules

- prioritize clarity and trust
- no decorative fake complexity
- keep visual restraint

## Deliverable

- stronger governance surfaces
- better record-to-governance continuity
- UI now matches the product’s trust model
