# Prompt 43 — Vehicle Detail Page, Payment, and Inquiry

Build the customer-facing vehicle detail page as the primary conversion surface for an individual unit.

## Goals

- create a premium vehicle detail page
- support conversion into inquiry, finance entry, save/favorite, and appointment actions
- show pricing/payment context clearly without confusion

## Tasks

1. Build the public unit detail page for `/shop/:unitId`.
2. Include at minimum:
   - primary unit hero/media area
   - key specs and highlights
   - pricing and status/availability
   - payment estimate summary or CTA
   - inquiry CTA
   - save/favorite CTA
   - compare CTA if available
   - appointment/test-drive CTA
3. Ensure the page is route-linked from the public inventory surface.
4. Add or update:
   - `/docs/ux/vehicle_detail_conversion_surface.md`

## Rules

- do not overload the page with dealership jargon
- do not misrepresent finance/payment as final numbers if they are estimates
- keep trust and clarity high

## Deliverable

- public vehicle detail page
- integrated inquiry/payment/appointment entry points
- docs for vehicle detail conversion design
