# Prompt 18 — Settings, Integrations, and Admin Surfaces

Turn settings pages from placeholders into real administrative surfaces for Outcome Dealer OS.

## Goals

- make roles and integrations pages real and useful
- expose current configuration and status instead of placeholder cards
- prepare the product for future operational configuration without overbuilding it now

## Tasks

1. Review current settings pages and shell entry points.
2. Build meaningful settings surfaces for:
   - role model overview
   - permission visibility
   - integration catalog or status surface
   - environment or configuration notes where appropriate
3. Make integrations settings explain current reality:
   - what is planned
   - what is connected
   - what is mock or pending
4. Add or update:
   - `/docs/architecture/settings_and_admin_surfaces.md`
   - `/docs/architecture/integration_surface_status.md`

## Rules

- do not invent fake integrations
- do not build a full admin console beyond current scope
- keep settings surfaces honest and useful

## Deliverable

- settings pages that are no longer placeholders
- clearer admin and integration surfaces
- docs for current admin and integration status
